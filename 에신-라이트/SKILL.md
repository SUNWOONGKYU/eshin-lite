# /에신-라이트 — 경량 에이전트 제조 공장

> 본체 **에신**(`/에신-llm-dependent-agent-create`): https://github.com/SUNWOONGKYU/llm-dependent-agent-create

> **라이트 = 사소·단순·저위험·로컬 1개 에이전트를 빠르게 만들고 자가검증으로 끝낸다.**
> "제대로·무겁게" 해야 하는 것(별도 Verification 에이전트·고위험 도메인·페르소나 토론·운영 감시·포털·배치)은 **라이트가 하지 않는다 — 그건 본체 `/에신`의 일이다.** 그런 신호가 보이면 만들다 멈추고 **본체로 라우팅**한다.
>
> ⚠️ **단, "라이트 = 막 만든다"가 아니다.** *싸고 필수적인 안전*(실호출 1회 검증·보안·BOM 안전진단·정직·승인·에스컬레이션 트리거)은 **그대로 한다.** 라이트가 줄인 것은 *무거운 검증 절차*이지 *기본 안전*이 아니다. 비중 있거나 위험하면 → 본체.

## ⚖️ 적용 대상 (L0에서 반드시 판정)

| 라이트로 OK | → 본체 `/에신`으로 (라이트 부적합) |
|---|---|
| 단일 책임·단순 입출력 · 1개 에이전트 | 여러 에이전트 묶음(배치) / 1호스팅=N(포털) |
| 외부 부품 거의 없음(기본 LLM 위주, 검증 부품 ≤2) | 검증 필요한 외부 부품 다수·복잡 인프라 |
| **저위험 전용** | **모든 고위험 도메인**(법률·의료·재테크·정치 등) — 면책·별도검증 필요 |
| 단발 · 단순 반복 · **가벼운 자가점검 루프(bounded ≤N)** | 복잡한 자율 루프·상태 머신·다중 에이전트 |
| 자가검증으로 충분한 *사소한* 산출물 | **비중 있는 산출물**(별도 Verification·엄격 검증 필요) |
| UI 없음(CLI·파일 산출) | UI·웹·배포형(접근성·클릭검증 필요) |

→ **하나라도 오른쪽이면 본체 `/에신`.** 진행 중 오른쪽 신호가 생기면(부품 3개+·포털화·배치화·고위험·UI·비중↑) 즉시 **본체로 에스컬레이션**.

## 🚫 라이트도 절대 생략 안 하는 것 (싸고 필수)
단일 책임 · **자기완결**(외부 스킬·외부 AI 의무 호출 금지) · **보안**(시크릿 하드코딩·`eval`/`exec`·검증 없는 shell 금지) · 종료·비용 한도(반복형) · **승인 게이트(L2)** · **"파일 생성 ≠ 동작함"**(실호출 1회 검증) · **BOM 안전진단**(라이선스 불명·시크릿·미유지 부품 컷) · **출력 정직**(과장·허위 금지) · **에스컬레이션 트리거**(무거우면 본체로 — 이게 있어야 아래 "본체로 미룸"이 안전하다).

## 🔼 라이트가 하지 않고 본체 `/에신`으로 미루는 것
- **별도 Verification 에이전트** — 라이트는 *자가* 실호출·정적 점검으로 종결("비중 있는 산출물은 별도 검증, **사소한 작업 제외**" 원칙과 일치). 별도 검증 주체가 필요할 만큼 비중 있으면 → 본체.
- **고위험 도메인 면책·출력정책 기계** — 라이트는 *저위험만* 만든다. 고위험이면 만들지 말고 → 본체.
- **깊은 검토**(페르소나 토론·다회 평가 루프·외부 LLM 이중검증) → 본체.
- **운영 감시 루프**(에러율·사용률 임계값 모니터) → 본체/별도. 라이트는 "만들고 끝"(아래 L6 한 줄 메모만).
- **포털·배치·UI·웹 배포** → 본체.

## 💳 LLM 호출 비용 원칙 (제작 ↔ 런타임 분리 — 반드시 지킴)
**"에이전트를 만드는 것"과 "만들어진 에이전트가 돌아가는 것"은 완전히 별개다. 인프라 A/B는 *런타임*에만 해당하지, 제작과는 무관하다.**
- **제작·검증 LLM = 항상 구독(Claude Code 본체).** 기획·조립·실호출 검증은 전부 로컬·구독으로. **A/B 불문, 제작·검증에 API 키·크레딧 사용 금지.** (검증하려고 API를 호출하면 무효 키가 API 모드로 끌려가 크레딧 한도에 막히는 사고가 난다.)
- **산출 에이전트의 런타임 LLM** = **B(로컬)→본인이 쓰면 `claude` CLI(구독 OAuth) 1순위**, 남에게 배포하면 그 사람 키. (포털 A·API 키 런타임은 본체 영역.)

## 공정 (L0 → L6) — 짧고 곧게

### L0. 시동 + 라이트 적합성 + 인프라
- 인자/맥락에서 제조 대상 도출. 위 적합성 표로 판정 — **부적합(고위험·포털·배치·UI·비중↑)이면 즉시 본체 안내하고 중단.**
- **인프라**: 항상 **B — 로컬 설치**(로컬 .py + LLM CLI). LLM은 **본인용 = `claude` CLI(구독) 기본**, API 키는 배포용만(💳 원칙).
- **설치 위치 = 바탕화면 `AI 에이전트` 폴더 안**(`Desktop\AI 에이전트\{name}\`). 본체(engine·run.bat 등)를 **바탕화면에서 바로 보이게** 둔다 — 사용자가 찾기 쉽게. **딥 경로(예: `C:\Dev\...`) 금지**(묻혀서 못 찾는 사고 방지).

### L1. 요구 발굴 — 핵심 문답 (간소)
정체성 요소를 덮되 **짧게**. 선택지 질문 우선, 모르면 "AI 추천".
- **★ Trivial Fast-Path**: 부품 0 · 저위험 · 단발 · 입출력 자명(예: 텍스트→텍스트 교정/요약/변환)이면 **문답 생략하고 한 줄 목표만 잡아 바로 L2(간이 승인)로.** (단순한 걸 질문으로 늘리지 말 것 — 시간은 코드가 아니라 과잉 문답에서 샌다.)
- 그 외: 목표·이름 / 입력·출력·트리거 / 유형·루프 / 도구·LLM·메모리 / 제약·실패모드 / 종료·보안·비용 을 **꼭 필요한 것만** 1~5문. **고위험 감지 시 → 본체로.**

### L2. 간이 목표서 ⛔ (승인 게이트)
```
━━ 에이전트 제조 목표서(라이트) — {이름} ━━
🎯 목표: (한 줄)
🏷️ 이름·분류 · 📂 유형 · 🏗️ 인프라 B(로컬)
🧱 죽이는 수작업(AS-IS→TO-BE): (1줄)
📊 KPI: ① 호출 동작(의도대로 작동) ② 사용자 여정 1개
🤖 LLM: claude CLI(구독)→폴백 · 🛑 종료 · 🔐 보안 · 💰 한도
🧩 부품(BOM): (있으면 1~2개 + 라이선스, 없으면 "0")
━━ 승인하시겠습니까? ━━
```
⛔ 승인 없이 이후 단계 금지. (Trivial Fast-Path도 이 한 줄 승인은 받는다.)

### L2.5. 빠른 도메인 스캔 (선택, 가볍게)
필요할 때만: "① 같은 걸 푸는 기존 자산 있나(차별점) ② 표준/함정 1~2개" — 웹 검색 2~3건, 3줄 요약. 자명하면 생략.

### L3. 발굴 + 간이 BOM 안전진단
- 기본 **LLM 단독**. 외부 부품이 꼭 필요할 때만 1~2개(공개저장소·OSS·공개 데이터). **데이터 API 부품 1개·잘 유지되는 라이브러리 1개는 라이트 범위(부품 ≤2).**
- **안전진단(필수·생략 금지)**: `eval`/`exec`·하드코딩 시크릿·deprecated·**라이선스 불명**·1년+미유지·보안/언어 규칙 위반 → **🔴 컷**(되살리지 않음). 3분류 🟢적용 / 🟡재단 후 / 🔴컷. (예: 라이선스 불명 라이브러리 🔴, 라이선스 명확한 공개 데이터 🟢.)

### L4. 제조 — 조립
- **system_prompt**(역할·동작·출력정책) + **에이전트 5대 요소 표준 구현**(아래) + **보안**·비용 한도.
  - **① 목표(Goal)** — system_prompt에 역할·출력 형식·규칙 명시
  - **② 두뇌(LLM)** — `claude` CLI(구독) 1순위 → `gemini` 폴백
  - **③ 도구(Tools)** — 입력 읽기·저장 등 실제 도구(해당 시). 없으면 "LLM 단독"임을 명시
  - **④ 메모리(Memory)** — 직전 결과 기억(작업기억, 수정 반복용) + 처리 이력 파일 누적(장기기억)
  - **⑤ 자율 자가점검 루프(Loop)** — 생성 직후 **출력 필수요소·요청 반영 여부를 스스로 점검해 부족하면 보완(bounded, 최대 N회·기본 3)**. ⚠️ *가벼운 자가점검까지가 라이트* — 복잡한 상태머신·다중 에이전트 루프는 본체.
- **결과 확인·저장 (필수, 생략 금지)** — "결과가 어디 있는지 못 찾는" 사고를 막는다.
  - **기본값 = 클릭 단계 없이 자동 저장.** 결과 파일을 에이전트 폴더에 바로 저장하고, 생성 완료 즉시 **기본 뷰어로 자동으로 연다**(`os.startfile(path)` — Windows). 저장 대화상자는 띄우지 않는다(2026-07-07 실전: 대화상자 클릭 1회가 반복 실습에서 누적 시간을 늘림 — 자동 오픈만으로 "어디 있는지 못 찾는" 문제는 이미 해결됨).
  - 다른 위치에 저장하고 싶으면 열린 결과 파일을 사용자가 직접 복사/이동하면 된다 — 이 정도는 에이전트가 대신할 필요 없음.
  - 콘솔에도 최종 저장 경로를 굵게/구분선으로 눈에 띄게 출력한다.
- 산출 = **로컬 설치형 데스크톱 에이전트**. 반드시:
  - `engine.py` — LLM 호출 본체. **기본값 = `claude` CLI(구독) 1순위 → `gemini` CLI 폴백.** API 직접호출은 배포용만. 구독 강제 = 호출 전 환경의 `ANTHROPIC_API_KEY`를 제거하는 헬퍼(`_sub_env`)로 호출(무효 키가 API 모드로 끌려가 막히는 사고 차단). CLI는 임시 디렉터리(`tempfile.gettempdir()`)를 `cwd`로(세션로그가 에이전트 폴더 오염 방지). 첫 줄 `sys.stdout/stdin.reconfigure(encoding="utf-8")`.
    ⚠️ **Windows에서 CLI 경로는 `shutil.which()`로 찾고, 프롬프트는 명령행 인자가 아니라 `subprocess.run(..., input=prompt_text)`로 stdin 전달할 것.** `claude`/`gemini`는 npm 설치 시 `.cmd` 배치파일이라 Windows가 내부적으로 cmd.exe를 거쳐 실행하는데, 프롬프트에 HTML 태그 등 `<`·`>`가 있으면 인자로 넘길 때 cmd.exe가 리다이렉션 기호로 오인해 내용이 깨진다(2026-07-07 실전 확인). 같은 이유로 `stdin=subprocess.DEVNULL`도 피하고(입력 자체를 stdin으로 주므로 불필요), 인자 전달은 아예 쓰지 않는다.
  - `run.bat` — `chcp 65001>nul` · `set PYTHONUTF8=1` · `set PYTHONIOENCODING=utf-8` · `pushd "%~dp0"` · `pause` · **한글 echo 금지**.
  - `install_shortcut.ps1` — **바탕화면 설치(필수)**. ⚠️ **UTF-8 BOM 저장**(PowerShell 5.1이 BOM 없는 UTF-8을 cp949로 오독 → 한글 .lnk 이름 깨짐/COM 에러). 본체 폴더와 바로가기를 **둘 다 `Desktop\AI 에이전트\` 안**에: 본체 `Desktop\AI 에이전트\{이름}\`, 바로가기 `Desktop\AI 에이전트\{이름}.lnk`. **바로가기 타깃 = `cmd.exe`, 인자 = `/c "run.bat"`**(전체 경로 따옴표, WorkDir=본체 폴더). 더블클릭 시 콘솔 창이 떠서 실행됨. ⚠️ **`conhost.exe`를 바로가기 타깃으로 직접 쓰지 말 것** — 실행 자체가 안 되는 환경이 있다.
- (슬래시 SKILL.md만 만드는 건 "터미널 안에서만 쓰는 도구"라고 명시할 때만 예외.)
- **DB·웹 호스팅·포털 산출물 없음.** 🔴 컷한 자리는 자체 작성.

### L5. 자가 검토 + 실호출 검사 (자가검증으로 종결)
> 라이트는 *별도 Verification 에이전트를 쓰지 않는다*. 자가 점검으로 끝낸다. (비중 있거나 고위험이면 여기서 멈추고 → 본체.)
- **정적**: 시크릿 누출·`eval`·deprecated·구조 점검.
- **동적·실호출(필수)**: **실제 1회 호출**로 의도대로 작동 + 산출물 형식 확인("파일 생성 ≠ 동작함"). ⚠️ Windows에서 PowerShell 파이프(`$x | python`)는 한글을 깨뜨릴 수 있음 → **UTF-8 파일 + `cmd /c "chcp 65001 & python engine.py < in.txt"`** 로 검증.
- **간단 자가평가(5기준 약식)**: 목적 충족 · 품질·정합(설명·메타가 기능과 일치) · 리스크(실패모드 대응·자기완결 위반 0) · 실행성 · 정직(과장 0). 미흡하면 L4로 1회 회귀.
- 결함 회귀: 문구→L4 / 부품→L3 / 정책·KPI→L2.
